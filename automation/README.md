# Web 自动化测试补充

本目录用于存放 OpenCart 项目的 Web 自动化测试相关材料。当前阶段先沉淀自动化测试范围、用例拆分和脚本目录规划；后续如果编写 Selenium、Playwright 或其他自动化脚本，可以继续放在本目录下。

## 建议目录

```text
automation/
├── README.md
├── test_login.py
├── test_product_search.py
├── test_cart.py
└── test_checkout.py
```

## 优先自动化场景

| 场景 | 自动化价值 | 原因 |
| --- | --- | --- |
| 用户登录 | 高 | 每次回归测试都需要验证账号入口是否正常 |
| 商品搜索 | 高 | 搜索是电商核心入口，适合做稳定回归 |
| 加入购物车 | 高 | 购物车连接商品浏览和结算流程 |
| 修改购物车数量 | 中 | 可覆盖数量边界和金额计算 |
| 下单结算主流程 | 高 | 覆盖电商核心业务闭环 |
| 后台登录 | 中 | 后台管理流程执行前置条件 |

## 自动化用例设计原则

- 优先选择稳定、重复执行价值高的主流程。
- 自动化脚本不替代探索性测试和复杂异常场景分析。
- 每条自动化用例只验证一个主要业务目标，避免脚本过长。
- 测试数据不要写死真实密码，建议使用环境变量或本地配置文件。
- 执行失败时保留截图、页面标题、当前 URL 和错误信息，方便定位问题。

## 可落地脚本方向

1. 登录成功后断言进入用户中心。
2. 搜索已存在商品后断言商品列表包含目标商品。
3. 商品详情页点击加入购物车后断言购物车数量变化。
4. 修改购物车数量后断言总价重新计算。
5. 完成下单流程后断言出现订单成功提示或订单号。

## 面试讲解口径

可以说明：本项目先用手工测试覆盖业务理解和用例设计，再挑选稳定的高频主流程做 Web 自动化回归测试。自动化重点不是追求用例数量，而是把登录、搜索、购物车、下单这些核心链路变成可重复执行的回归检查。
