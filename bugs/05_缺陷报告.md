## BUG_001 MacBook 商品结算时无可选支付方式

| 字段 | 内容 |
| --- | --- |
| 缺陷编号 | BUG_001 |
| 缺陷标题 | MacBook 商品结算时无可选支付方式 |
| 所属模块 | 下单结算 |
| 发现环境 | Windows + Edge，本地 OpenCart 测试环境 |
| 严重程度 | 一般 |
| 优先级 | P2 |
| 前置条件 | 用户已登录，购物车中存在 MacBook 商品，已填写账单地址 |
| 复现步骤 | 1. 将 MacBook 加入购物车；2. 进入结算页；3. 填写账单信息；4. 进入 Payment Method 步骤 |
| 实际结果 | Payment Method 区域未展示任何可选支付方式，点击 Continue 后提示 Payment method required，用户无法继续完成下单 |
| 预期结果 | 系统应展示可用支付方式，或在商品/结算页明确提示该商品当前无法完成支付结算 |
| 截图证据 | evidence/screenshots/BUG_001_checkout_payment_method_missing.png |
| 状态 | 新建 |

## BUG_002 订单提交后邮件警告导致前端出现 JSON 解析错误弹窗

| 字段 | 内容 |
| --- | --- |
| 缺陷编号 | BUG_002 |
| 缺陷标题 | 订单提交后邮件警告导致前端出现 JSON 解析错误弹窗 |
| 所属模块 | 下单结算 |
| 发现环境 | Windows + Edge，本地 OpenCart 测试环境 |
| 严重程度 | 一般 |
| 优先级 | P2 |
| 前置条件 | 用户已登录，购物车中存在 iPhone 商品，已填写账单地址和配送地址 |
| 复现步骤 | 1. 进入结算页；2. 选择 Flat Shipping Rate；3. 选择 Cash On Delivery；4. 勾选 Terms & Conditions；5. 点击确认订单 |
| 实际结果 | 点击确认订单后，页面弹出 SyntaxError，提示 mail() 无法连接 localhost:25，短暂停留在 Loading 状态；随后订单成功生成并跳转到成功页面 |
| 预期结果 | 订单提交后应直接跳转到订单成功页面；即使邮件发送失败，也不应将 PHP Warning 输出到前端 JSON 响应中 |
| 截图证据 | evidence/screenshots/BUG_002_order_submit_mail_json_error.png |
| 状态 | 新建 |

## BUG_003 购物车商品数量输入负数后购物车被清空且无明确提示

| 字段 | 内容 |
| --- | --- |
| 缺陷编号 | BUG_003 |
| 缺陷标题 | 购物车商品数量输入负数后购物车被清空且无明确提示 |
| 所属模块 | 购物车 |
| 发现环境 | Windows + Edge，本地 OpenCart 测试环境 |
| 严重程度 | 一般 |
| 优先级 | P2 |
| 前置条件 | 购物车中已存在商品 |
| 复现步骤 | 1. 进入购物车页面；2. 将商品数量修改为 -1；3. 点击更新按钮 |
| 实际结果 | 购物车被直接清空，页面提示 Your shopping cart is empty，未对负数数量给出明确错误提示 |
| 预期结果 | 系统应拒绝负数数量输入，并提示商品数量必须为正整数，不应直接清空购物车 |
| 截图证据 | evidence/screenshots/BUG_003_cart_negative_quantity.png |
| 状态 | 新建 |

## RISK_005 storage 目录位于 Web 目录内存在潜在安全风险

| 字段 | 内容 |
| --- | --- |
| 缺陷编号 | RISK_005 |
| 缺陷标题 | storage 目录位于 Web 目录内存在潜在安全风险 |
| 所属模块 | 后台安全配置 |
| 发现环境 | Windows + Edge，本地 OpenCart 测试环境 |
| 严重程度 | 建议 |
| 优先级 | P3 |
| 前置条件 | 管理员已登录后台 |
| 复现步骤 | 1. 打开后台；2. 触发 Important Security Notification 弹窗；3. 查看 storage 目录移动提示 |
| 实际结果 | 后台提示 storage directory should be moved outside of the web directory |
| 预期结果 | 正式环境中 storage 目录应位于 Web 目录之外，降低敏感文件被直接访问的风险 |
| 截图证据 | evidence/screenshots/RISK_005_storage_directory_security_notice.png |
| 状态 | 新建 |
