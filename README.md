# OpenCart 开源电商系统测试项目

这是一个基于 OpenCart 3.0.5.0 的电商测试练习项目。v1 阶段先完成手工测试执行，保留测试计划、测试点分析、测试用例、缺陷记录、截图证据和测试总结；v2 阶段在此基础上补充 Web 自动化测试和 Postman 接口测试设计，用来说明自己从手工测试继续向接口测试、自动化测试扩展的学习过程。

## 项目概况

| 项目 | 内容 |
| --- | --- |
| 被测系统 | OpenCart 3.0.5.0 |
| 项目类型 | 电商业务测试实践 |
| 测试环境 | Windows + Edge，本地 XAMPP + OpenCart |
| 测试范围 | 注册登录、商品浏览、购物车、下单结算、后台商品管理、后台订单管理、后台安全配置 |
| 测试方法 | 场景法、等价类划分、边界值分析、错误推测法、接口测试设计、自动化回归设计 |
| 管理工具 | XMind、Markdown、禅道、Postman、Git |

## 主要测试范围

| 模块 | 测试重点 |
| --- | --- |
| 前台注册登录 | 注册、登录、退出、密码错误、必填校验 |
| 商品浏览 | 分类、搜索、商品详情、价格、库存 |
| 购物车 | 加购、修改数量、删除商品、库存边界 |
| 下单结算 | 地址、配送方式、支付方式、订单确认 |
| 后台商品管理 | 新增商品、编辑商品、上下架、库存维护 |
| 后台订单管理 | 订单查询、订单详情、订单状态流转 |
| 客户管理 | 客户信息、客户组、客户审批 |
| 异常与兼容性 | 非法输入、重复提交、权限控制、浏览器兼容 |
| Web 自动化测试设计 | 登录、搜索、购物车、下单主流程的回归场景 |
| Postman 接口测试设计 | 登录、商品查询、购物车、订单相关接口的请求和断言 |

## 典型问题

- MacBook 商品进入结算后无可选支付方式，导致用户无法继续下单。
- 订单提交过程中邮件发送警告被输出到 JSON 响应，前端出现解析错误弹窗。
- 购物车商品数量输入 `-1` 后购物车被清空，页面未给出明确的正整数校验提示。
- 后台存在 install 目录和 storage 目录安全提示，说明本地部署仍有安全配置风险。

## 项目价值

这个项目保留了从测试分析到执行复盘的完整材料：先拆解业务流程，再设计用例，随后执行主流程并记录缺陷。文档中的用例、缺陷和截图可以相互对应，方便面试时解释测试思路、缺陷判断和项目整理过程。后续补充的 Web 自动化和 Postman 接口测试内容目前定位为“设计补充”，不写成已执行结果；等脚本、Postman Collection 和报告补齐后，再升级为可执行的自动化/API 测试成果。
