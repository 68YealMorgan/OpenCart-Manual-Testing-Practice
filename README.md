# OpenCart Testing Practice

本仓库用于沉淀基于开源电商系统 OpenCart 的测试项目材料。项目以手工测试为基础，补充 Web 自动化测试和 Postman 接口测试设计，目标是形成一套可用于测试实习投递和面试讲解的作品。

## 项目定位

- 测试对象：OpenCart 开源电商系统
- 项目类型：电商业务测试实践
- 测试范围：手工功能测试、Web 自动化回归测试设计、Postman 接口测试设计
- 主要工具：Xmind、Typora、Postman、Web 自动化测试工具、Git、GitHub、Gitee
- 重点能力：需求分析、测试点设计、测试用例设计、接口测试、自动化回归思路、缺陷记录、测试总结、项目复盘

## 核心测试范围

| 模块 | 测试重点 |
| --- | --- |
| 前台注册登录 | 注册、登录、退出、密码错误、必填校验 |
| 商品浏览 | 分类、搜索、商品详情、价格、库存 |
| 购物车 | 加购、修改数量、删除商品、库存边界 |
| 下单结算 | 地址、配送方式、支付方式、订单确认 |
| 后台商品管理 | 新增商品、编辑商品、上下架、库存维护 |
| 后台订单管理 | 订单查询、订单详情、订单状态流转 |
| 客户管理 | 客户信息、客户组、客户审批 |
| 异常与兼容性 | 非法输入、重复提交、权限控制、浏览器兼容 |
| Web 自动化测试 | 注册登录、商品搜索、购物车、下单主流程的回归测试设计 |
| Postman 接口测试 | 登录、商品查询、购物车、订单相关接口的请求设计与断言 |

## 交付物

- Xmind 测试点脑图 1 份
- Xmind 文件路径：`assets/opencart-test-points-final.xmind`
- Typora Markdown 文档 8 份以上
- 手工测试用例不少于 60 条
- Web 自动化测试补充设计 1 份
- Postman 接口测试补充设计 1 份
- 缺陷报告不少于 5 条，若实际缺陷较少，可补充“疑似缺陷/体验问题/风险点”
- 关键执行截图不少于 10 张
- GitHub 与 Gitee 仓库链接各 1 个

## 项目亮点写法

本项目围绕 OpenCart 开源电商系统，覆盖用户注册登录、商品浏览、购物车、下单结算、后台商品管理、订单管理、客户管理等核心电商业务流程。项目第一阶段使用 Xmind 和 Markdown 完成手工测试分析、用例设计、缺陷报告和测试总结；在此基础上补充 Web 自动化回归测试和 Postman 接口测试设计，用于体现从手工测试到接口测试、自动化测试的扩展能力。
