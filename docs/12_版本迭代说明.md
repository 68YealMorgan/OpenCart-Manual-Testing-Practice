# 12 版本迭代说明

这个文件用来说明项目是怎么一步步升级的。面试官如果看仓库，可以直接看到每个版本对应的能力变化。

## v1.0 手工测试执行版

时间：2026-06

这一版的重点是把手工测试流程做完整，而不是只写模板。

主要内容：

- 本地 XAMPP 环境搭建 OpenCart 3.0.5.0。
- 梳理前台注册登录、商品搜索、购物车、下单结算、后台商品和订单管理等测试范围。
- 使用场景法、等价类划分、边界值分析和错误推测法设计测试用例 29 条。
- 执行核心流程用例 11 条，其中通过 10 条，失败 1 条。
- 记录缺陷/风险点 5 项，并保留 14 张截图证据。
- 输出 README、测试计划、测试点分析、测试用例、缺陷报告、测试总结和简历描述。

这一版可以证明：我能把一个开源系统当作真实业务系统来拆解、设计用例、执行测试、记录问题和复盘。

## v2.0 测试能力扩展版

时间：2026-06

这一版是在学完 Web 自动化测试和 Postman 接口测试后补充的。重点是把原来的手工测试项目继续往接口测试、自动化测试和缺陷管理记录方向扩展。

主要内容：

- 新增 `automation/README.md`，整理 Web 自动化测试目录规划和优先自动化场景。

- 新增 `docs/10_Web自动化测试补充.md`，把登录、搜索、购物车、下单拆成候选自动化回归场景。
- 新增 `docs/11_Postman接口测试补充.md`，把登录、商品、购物车、订单拆成候选接口测试范围。

- 更新 `testcases/04_测试用例.md`，补充自动化测试和接口测试候选用例。
- 更新 `docs/07_简历描述.md`，加入自动化/接口测试方向写法，同时保留“设计补充”和“已执行”的边界。

这一版可以证明：我不是只停留在手工测试文档，而是在尝试把核心业务流程沉淀成可回归、可接口验证、可追踪缺陷状态的测试资产。

## 后续落地计划

后续不建议继续只加文档，应该补真实可执行材料。当前仍归在 v2 的后续完善范围内，不单独升级版本号。

优先级：

1. 用 Postman 抓取并整理登录、商品查询、购物车、订单相关请求，导出脱敏后的 Collection。
2. 使用 Selenium 或 Playwright 先写登录、商品搜索、加入购物车 3 条冒烟脚本。
3. 在禅道中补齐真实缺陷编号，并保存缺陷列表或详情页截图。
4. 保存自动化执行截图或报告。
5. 再更新简历，把“设计补充”改成“已执行并保留报告”。
