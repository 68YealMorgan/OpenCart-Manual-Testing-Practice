# 09 项目发布检查清单

## 仓库内容

- README 能说明项目背景、测试范围、执行结果和典型问题。
- `docs/` 中保留项目介绍、测试计划、测试点分析、测试总结和简历描述。
- `testcases/04_测试用例.md` 中的用例编号、步骤、预期结果和执行状态清晰。
- `bugs/05_缺陷报告.md` 中的缺陷标题、复现步骤、实际结果、预期结果和截图路径完整。
- `evidence/screenshots/` 中的截图不包含真实密码、个人联系方式或无关软件界面。
- `assets/` 中保留最终版测试点脑图和可追溯的脑图源文件。

## 发布前自查

| 检查项 | 状态 |
| --- | --- |
| README 已从计划口吻改为成果口吻 | 已完成 |
| 测试用例数量与执行统计一致 | 已完成 |
| 缺陷报告中的截图路径可对应到实际文件 | 已完成 |
| 禅道缺陷记录已去除工具生成提示 | 已完成 |
| Git 本地提交已完成 | 待完成 |
| GitHub/Gitee 远程仓库已配置 | 待完成 |

## 发布建议

仓库建议保持公开，方便投递简历时直接附链接。简历中不需要写过多工具名，重点写清楚测试对象、测试范围、用例数量、执行数量、缺陷数量和自己负责的工作。

推荐仓库名：

```text
OpenCart-Manual-Testing-Practice
```

推荐提交信息：

```text
docs: add opencart manual testing practice
```
