# 09 项目发布检查清单

## 仓库内容

- README 能说明项目背景、测试范围、执行结果和典型问题。
- `docs/` 中保留项目介绍、测试计划、测试点分析、测试总结和简历描述。
- `docs/10_Web自动化测试补充.md` 和 `docs/11_Postman接口测试补充.md` 已说明自动化/API 扩展边界。
- `docs/12_版本迭代说明.md` 已说明 v1 手工测试执行版和 v2 测试能力扩展版。
- `testcases/04_测试用例.md` 中的用例编号、步骤、预期结果和执行状态清晰。
- `bugs/05_缺陷报告.md` 中的缺陷标题、复现步骤、实际结果、预期结果和截图路径完整。
- `evidence/zentao/README.md` 中已按禅道字段整理缺陷记录，后续可补充禅道截图。
- `evidence/screenshots/` 中的截图不包含真实密码、个人联系方式或无关软件界面。
- `assets/` 中保留最终版测试点脑图和可追溯的脑图源文件。

## 发布建议

仓库建议保持公开，方便投递简历时直接附链接。简历中不需要写过多工具名，重点写清楚测试对象、测试范围、用例数量、执行数量、缺陷数量和自己负责的工作。

v2 版本发布时，建议强调“补充 Web 自动化测试设计和 Postman 接口测试设计”，不要写成已经完成大量自动化执行。等后续补齐脚本、Postman Collection 和执行报告后，再更新为 v3。

推荐仓库名：

```text
OpenCart-Manual-Testing-Practice
```

推荐提交信息：

```text
docs: add testing growth version notes
```
