# 01 项目介绍

## 项目背景

OpenCart 是一个开源电商系统，具备前台商品浏览、购物车、下单结算，以及后台商品、订单、客户等管理能力。本项目选择 OpenCart 作为测试对象，模拟真实电商业务测试流程，完成手工测试分析、用例设计、缺陷记录和测试总结，并在此基础上补充 Web 自动化测试与 Postman 接口测试设计。

## 项目目标

- 理解电商系统核心业务流程
- 掌握手工测试项目的完整产出链路
- 使用 Xmind 输出测试点脑图
- 使用 Typora 输出项目文档、测试用例、缺陷报告和总结
- 补充 Web 自动化测试范围，挑选适合回归的核心流程
- 补充 Postman 接口测试范围，设计接口请求、环境变量和断言
- 将项目整理到 GitHub 与 Gitee，作为实习投递展示材料

## 测试对象

- 项目名称：OpenCart
- 项目地址：https://github.com/opencart/opencart
- 系统类型：开源电商系统
- 主要用户角色：
  - 游客
  - 注册用户
  - 后台管理员

## 测试范围

| 范围 | 是否纳入 | 说明 |
| --- | --- | --- |
| 前台注册登录 | 是 | 覆盖基础账号流程 |
| 商品浏览与搜索 | 是 | 覆盖电商入口流程 |
| 购物车 | 是 | 覆盖购买前核心流程 |
| 下单结算 | 是 | 覆盖主业务闭环 |
| 后台商品管理 | 是 | 覆盖商品维护能力 |
| 后台订单管理 | 是 | 覆盖订单处理能力 |
| 客户管理 | 是 | 覆盖客户维护能力 |
| 支付真实扣款 | 否 | 不进行真实支付 |
| 性能测试 | 否 | 本项目聚焦手工功能测试 |
| Web 自动化测试 | 是，设计补充 | 作为手工测试后的回归扩展，优先覆盖登录、搜索、购物车、下单 |
| Postman 接口测试 | 是，设计补充 | 作为接口测试扩展，优先覆盖登录、商品、购物车、订单相关接口 |

## 项目产出

- 测试计划
- 测试点脑图
- 测试用例
- 缺陷报告
- 测试总结
- Web 自动化测试补充说明
- Postman 接口测试补充说明
- 简历描述
- GitHub/Gitee 展示仓库
