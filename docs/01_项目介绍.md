# 01 项目介绍

## 项目背景

OpenCart 是一个开源电商系统，具备前台商品浏览、购物车、下单结算，以及后台商品、订单、客户等管理能力。本项目选择 OpenCart 作为测试对象，模拟真实电商业务测试流程。v1 阶段完成手工测试分析、用例设计、缺陷记录和测试总结；v2 阶段在已有执行材料上补充 Web 自动化测试和 Postman 接口测试设计。

## 项目目标

- 理解电商系统核心业务流程
- 掌握手工测试项目的完整产出链路
- 使用 Xmind 输出测试点脑图
- 使用 Markdown 输出项目文档、测试用例、缺陷报告和总结
- 使用禅道记录缺陷，模拟测试团队中的缺陷管理流程
- 补充 Web 自动化测试范围，挑选适合回归的核心流程
- 补充 Postman 接口测试范围，设计请求范围、环境变量和基础断言
- 将项目整理到 GitHub 与 Gitee，作为实习投递展示材料

## 测试对象

- 项目名称：OpenCart
- 项目地址：https://github.com/opencart/opencart
- 系统类型：开源电商系统
- 主要用户角色：
  - 游客
  - 注册用户
  - 后台管理员

## 个人收获

通过本项目，我对电商系统的前台购物闭环和后台管理流程有了更完整的理解，能够将注册登录、商品搜索、购物车、下单结算、订单管理、商品管理等业务拆解为可执行的测试点和测试用例。

在测试设计方面，本项目结合了场景法、等价类划分、边界值分析和错误推测法。例如，主流程用场景法串联用户购买链路，购物车数量使用边界值和异常输入进行验证，后台订单和商品管理则重点关注数据展示、筛选和状态一致性。

在测试执行过程中，我记录了支付方式缺失、订单提交时邮件警告导致前端 JSON 解析弹窗、购物车负数数量处理不明确、后台安全配置提示等缺陷和风险点，进一步理解了缺陷报告中复现步骤、实际结果、预期结果和截图证据的重要性。

通过整理 README、测试计划、测试用例、缺陷报告、测试总结、截图证据和简历描述，本项目也帮助我熟悉了一个测试作品从执行、记录、复盘到公开展示的完整流程。后续补充自动化和接口测试设计时，我也进一步意识到：手工测试用例不是孤立文档，稳定的主流程可以沉淀为自动化回归用例，关键业务请求也可以用 Postman 做接口级验证。
