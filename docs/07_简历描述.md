## 面试讲解思路

1. 为什么选 OpenCart：它是开源电商系统，业务链路比较完整，适合练习注册、购物车、下单和后台管理等测试场景。
2. 测试范围：重点覆盖前台购物闭环和后台商品/订单管理流程。
3. 用例设计方法：主流程用场景法串起来，输入项用等价类和边界值，异常情况用错误推测法补充。
4. 典型缺陷：可以重点讲“订单提交邮件警告导致 JSON 解析错误弹窗”和“购物车数量输入 -1 后被清空”。
5. 项目收获：熟悉了手工测试从测试分析、用例设计、执行截图、缺陷记录到测试总结的完整流程。
