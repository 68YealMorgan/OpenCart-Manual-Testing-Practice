## 测试结论

- 前台注册、登录、商品搜索、商品详情查看、加入购物车、修改购物车数量、iPhone 商品下单、后台查看订单、后台商品列表筛选等主流程能够跑通。
- 商品浏览、后台订单查询、后台商品列表展示等基础功能表现较稳定，能够满足核心验证需求。
- 下单结算和购物车边界输入存在一定风险，例如部分商品结算时无可选支付方式、负数数量导致购物车被清空且提示不明确。
- 后台存在安装目录和 storage 目录安全提示，说明本地环境仍有安全配置项未完全处理，正式部署前需要重点检查。
- 优先建议修复订单提交时邮件警告污染 JSON 响应、购物车数量边界校验不明确等问题。

## 典型问题分析

| 问题 | 影响 | 建议 |
| --- | --- | --- |
| MacBook 商品结算时无可选支付方式 | 用户无法继续完成下单，容易误以为系统故障 | 检查商品配送属性与支付方式适用条件；在无可用支付方式时给出明确说明 |
| 订单提交后出现邮件警告导致 JSON 解析弹窗 | 订单虽能生成，但前端出现技术错误弹窗，影响用户信任 | 邮件发送失败不应输出 PHP Warning 到 JSON 响应；应记录日志并返回规范结果 |
| 购物车数量输入 -1 后购物车被清空 | 用户输入异常值后缺少明确校验提示，可能造成误删除体验 | 对购物车数量做正整数校验，并在页面显示明确错误提示 |
| 后台存在 install 目录安全提示 | 正式环境可能存在安装目录暴露风险 | 删除或隔离 install 目录，并在部署流程中加入安全检查 |
| storage 目录位于 Web 目录内 | 敏感文件存在被 Web 直接访问的潜在风险 | 将 storage 目录迁移到 Web 根目录之外 |

## 复盘

- 通过本项目熟悉了电商系统从注册登录、商品浏览、购物车、下单到后台订单管理的完整业务闭环。
- 在用例设计中练习了场景法、等价类划分、边界值分析和错误推测法，能够把业务流程拆解为可执行、可验证的测试步骤。
- 在执行过程中学习了如何区分功能缺陷、环境风险和体验问题，并使用统一模板记录缺陷标题、复现步骤、实际结果、预期结果和截图证据。
- 通过整理 README、测试计划、测试用例、缺陷报告、测试总结和截图目录，完成了一套可公开展示的测试项目材料。
- 在学习 Web 自动化测试和 Postman 接口测试后，进一步把登录、商品搜索、购物车、下单等主流程拆成可回归的自动化候选场景，并整理了接口测试的请求范围、环境变量和基础断言思路。
